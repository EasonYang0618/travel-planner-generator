Problem statement: Design and implement a maintainable web application for a small travel agency that generates personalized, day-by-day city travel itineraries. The system must expose a Flask REST API that accepts destination, trip length, budget level, and user interests and returns a structured itinerary, and a responsive website that lets users submit those parameters and view a clear, printable day-by-day plan. Non-functional requirements: simple, user‑friendly presentation, basic input validation and error handling, deployable on modest infrastructure, and easy extensibility for future POI/data-source integrations.
