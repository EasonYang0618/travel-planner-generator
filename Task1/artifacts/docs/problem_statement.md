Build a deterministic Flask-based web application and single-page website that generates personalized multi-day city travel itineraries: the Flask API must expose GET /health and POST /api/itinerary (accepting destination, days, budget, interests, travel_style) and return JSON with top-level keys destination, days, budget, interests, travel_style, overview, itinerary, and tips; itinerary must be an array of length days where each day includes day, morning, afternoon, evening, and budget_note. Activities must be produced by expanding each user interest into several concrete activity options and rotating morning/afternoon/evening slots using destination, day number, time slot and interests so items do not repeat the same generic fallback; activity titles/descriptions must be human-readable strings (activity objects may exist but the frontend must render readable fields and never display [object Object]); estimated costs must be numeric and adjusted by budget. Invalid input must return HTTP 400 with a JSON error object. The frontend must not create an image preview area and must use the stable element IDs plannerForm, destination, days, budget, interests, travel_style, resultsContainer, daysContainer, formMessage, errorMessage and stable functions collectFormData, validateFormData, setLoading, showError, renderItinerary, renderDay, renderActivity, and formatActivityText. Deliver stable, deterministic Flask API and website artefacts suitable for automated pytest checks and Docker deployment.
