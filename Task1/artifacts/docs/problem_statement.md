Build a deterministic web application (Flask API + website) that generates personalised city travel itineraries for a given destination, days, budget, interests, and travel_style and produces stable, testable artefacts suitable for automated pytest checks and Docker deployment. The Flask API must expose GET /health and POST /api/itinerary (POST accepts destination, days, budget, interests, travel_style) and return HTTP 400 with a JSON error object for invalid input. A successful POST must return a JSON with top-level fields destination, days, budget, interests, travel_style, overview, itinerary, and tips; itinerary must be a list whose length equals days and every day must include day, morning, afternoon, evening, and budget_note. Activities must be generated by first expanding each interest into several concrete activities (so multi-day trips do not visibly repeat the same few items), rotating morning/afternoon/evening items using destination, day number, time slot, and interests (no repeating the same generic fallback each day), and giving human-readable string titles/descriptions (no raw objects like [object Object]); estimated costs must be numeric values adjusted by budget. The generated frontend must not create an image preview area, must render readable activity fields (frontend may accept activity objects but must never display [object Object]), and must include stable DOM IDs: plannerForm, destination, days, budget, interests, travel_style, resultsContainer, daysContainer, formMessage, errorMessage and stable functions: collectFormData, validateFormData, setLoading, showError, renderItinerary, renderDay, renderActivity, formatActivityText. All code should be deterministic and stable so the produced Flask API and website artefacts can be automatically tested and deployed in Docker.
