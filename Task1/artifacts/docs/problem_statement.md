Build a deterministic, testable web application (Flask backend + single-page website) that generates personalised city travel itineraries for a small travel agency; the software must produce stable Flask API and website artefacts suitable for automated pytest checks and Docker deployment. The Flask API must expose GET /health and POST /api/itinerary, where POST accepts destination, days, budget, interests, and travel_style and returns HTTP 400 with a JSON error object for invalid input. A successful POST must return top-level keys destination, days, budget, interests, travel_style, overview, itinerary, and tips; itinerary must be a list with length equal to days and each day an object containing day, morning, afternoon, evening, and budget_note. Activities must be human-readable strings (no raw dictionaries or [object Object]); estimated costs must be numeric and adjusted by budget. Generate varied, non-repetitive activities by expanding each interest into several concrete activities (e.g., food → breakfast street-food lane, local market tasting, signature restaurant meal, dessert stop, evening snack) and rotate morning/afternoon/evening items using destination, day number, time slot, and interests so each day has a distinct theme where possible. The frontend must not create an image preview area, must render activity fields readable, use the stable element IDs plannerForm, destination, days, budget, interests, travel_style, resultsContainer, daysContainer, formMessage, errorMessage, and implement stable functions collectFormData, validateFormData, setLoading, showError, renderItinerary, renderDay, renderActivity, and formatActivityText.
