Build a deterministic, testable web application (Flask backend + static frontend) that generates personalised day-by-day city travel itineraries for tourists and produces stable artefacts suitable for automated pytest checks and Docker deployment. The Flask API must expose GET /health and POST /api/itinerary (accepting destination, days, budget, interests, travel_style) and must return HTTP 400 with a JSON error object for invalid input. A successful POST must return a JSON object with top-level destination, days, budget, interests, travel_style, overview, itinerary, and tips; itinerary must be a list of length = days, where each day contains day, morning, afternoon, evening, and budget_note. Activities must be human-readable strings (no raw dictionaries or “[object Object]”), may include dictionaries internally but the frontend must render readable fields, and estimated costs must be numeric values scaled by budget. The generator must expand each interest into several concrete activities and rotate morning/afternoon/evening items using destination, day number, time slot, and interests so days do not repeat the same generic fallback; each day should have a different theme where possible. The frontend must not create its own image preview area and must use stable DOM IDs (plannerForm, destination, days, budget, interests, travel_style, resultsContainer, daysContainer, formMessage, errorMessage) and stable functions (collectFormData, validateFormData, setLoading, showError, renderItinerary, renderDay, renderActivity, formatActivityText). All code must be deterministic and stable for automated testing and Docker deployment.
