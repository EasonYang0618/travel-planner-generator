Build a deterministic Flask + frontend web application for a travel agency that generates personalised city itineraries and produces stable, testable artefacts suitable for automated pytest checks and Docker deployment. The Flask API must expose GET /health and POST /api/itinerary; POST /api/itinerary must accept destination, days, budget, interests, and travel_style and return HTTP 400 with a JSON error object on invalid input. A successful response must include top-level destination, days, budget, interests, travel_style, overview, itinerary, and tips; itinerary must be a list whose length equals days, and each day must include day, morning, afternoon, evening, and budget_note. Activity titles/descriptions must be human-readable strings (never raw objects) and estimated costs must be numeric values adjusted by budget. The generator must expand each user interest into several concrete activities (e.g., food → breakfast street-food lane, market tasting, signature restaurant, dessert stop; outdoor → lakefront walk, city park, scenic viewpoint) so multi-day plans do not repeat the same generic fallback; rotate morning/afternoon/evening items using destination, day number, time slot, and user interests, and give each day a distinct theme where possible. The frontend must not create its own image preview area, must render readable fields (no [object Object]), use stable element IDs plannerForm, destination, days, budget, interests, travel_style, resultsContainer, daysContainer, formMessage, errorMessage, and expose stable functions collectFormData, validateFormData, setLoading, showError, renderItinerary, renderDay, renderActivity, and formatActivityText. Code must be deterministic and stable for automated testing and Docker deployment.
